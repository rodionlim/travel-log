# 2026-05

## 2026-05-18

- Itinerary items now support optional 1-10 ratings, selected-day or whole-trip filtering by rating threshold (`>=` / `<=`), and a new `FOOD` item type in the day view and item form.
- Added itinerary item `rating` persistence and sync support, including a Room migration to database version 9 and a trip sync protocol bump to 3 because the itinerary item contract now includes ratings and the `FOOD` enum value.
- Added a `SHOPPING` itinerary item type to the day view, item form, linked-budget mapping, and filters, and bumped trip sync protocol to 4 because the shared itinerary item enum expanded again.
- Added a `GROCERIES` itinerary item type to the day view, item form, linked-budget mapping, and filters, and bumped trip sync protocol to 5 because the shared itinerary item enum expanded again.
- Budget expenses now support an optional manual date in the add/edit form, prefill that date on edit, and persist it to `Expense.date`.
- Budget now has a day-by-day view with primary day chips, grouped daily spend sections, an `Unscheduled` bucket for undated expenses, and a secondary category filter scoped through the current day selection.
